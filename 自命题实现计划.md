# 自命题实现计划：异常波动样本的建模与误差分析

## 一、任务定位

本自命题建立在 Kaggle 官方短期价格变化回归任务之上，不重新定义预测目标，而是进一步分析模型在不同波动强度样本上的表现差异。

题目名称：

```text
加密货币短期价格预测中异常波动样本的建模与误差分析
```

核心思想：

```text
整体 Pearson 较高的模型，不一定能较好预测大幅涨跌样本。
因此需要把验证集按 abs(label) 划分为普通波动、中等波动和异常波动样本，
分别评估误差，并尝试通过样本加权训练改善异常波动样本预测效果。
```

## 二、基本假设与成功标准

### 2.1 基本假设

1. 训练集中的 `label` 表示未来短期价格变化，`abs(label)` 可以作为样本波动强度的近似度量。
2. 绝大多数样本的 `label` 接近 0，少数样本存在较大正向或负向波动。
3. 异常波动样本数量较少，但可能对 RMSE、MAE 等误差指标产生较大影响。
4. 样本加权训练可能提高模型对异常波动样本的关注程度，但也可能牺牲普通样本或整体 Pearson。
5. 实验仍然采用时间顺序划分，避免随机划分导致未来信息泄露。

### 2.2 成功标准

本任务不以 Kaggle 榜单分数提升作为唯一成功标准，而以能否回答以下问题作为成功标准：

1. 能清楚定义异常波动样本，并给出各组样本数量和占比。
2. 能比较 Ridge、普通 LightGBM、加权 LightGBM 在各波动区间上的表现。
3. 能说明异常波动样本是否是主要误差来源之一。
4. 能判断加权训练是否改善异常波动样本表现，以及是否带来整体性能损失。
5. 能形成可放入论文和 PPT 的表格、图表和结论。

## 三、研究问题

建议在报告中围绕以下三个问题展开：

1. 不同模型在普通波动、中等波动和异常波动样本上的预测误差是否存在明显差异？
2. 异常波动样本虽然数量较少，是否对整体 RMSE、MAE 等误差指标有较大贡献？
3. 对异常波动样本赋予更高训练权重后，模型在异常波动区间内的预测表现是否改善？

## 四、样本分组方案

### 4.1 波动强度定义

对每个样本定义波动强度：

```text
volatility_i = abs(label_i)
```

### 4.2 推荐分组

在验证集上按 `abs(label)` 的分位数划分样本：

| 分组 | 定义 | 含义 |
|---|---|---|
| 普通波动样本 | `abs(label)` <= 80% 分位数 | 大多数普通行情 |
| 中等波动样本 | 80% 分位数 < `abs(label)` <= 95% 分位数 | 波动较明显的行情 |
| 异常波动样本 | `abs(label)` > 95% 分位数 | 大幅涨跌样本 |

如果实验时间不足，可以退化为两组：

| 分组 | 定义 |
|---|---|
| 普通样本 | `abs(label)` <= 90% 分位数 |
| 异常波动样本 | `abs(label)` > 90% 分位数 |

但报告中更推荐使用三组，因为分析层次更清楚。

## 五、模型设计

### 5.1 对比模型

建议至少包含以下三个模型：

| 模型 | 作用 |
|---|---|
| Ridge baseline | 线性基准模型，用于观察简单线性模型在异常样本上的表现 |
| LightGBM baseline | 当前主力非线性模型，用于代表官方预测任务主要方法 |
| Weighted LightGBM | 新增自命题模型，用于检验加权训练能否改善异常波动样本预测 |

如果时间允许，可以加入：

| 模型 | 作用 |
|---|---|
| 时序扩展 LightGBM | 检查 lag、rolling、diff 等时序派生特征是否改善异常波动样本表现 |
| CatBoost + XGBoost 融合 | 作为复杂树模型对照，分析本地高分模型是否真的更擅长异常样本 |

### 5.2 加权 LightGBM 方案

训练时根据训练集 `abs(label)` 分位数设置样本权重：

| 训练样本类型 | 权重 |
|---|---:|
| 普通波动样本 | 1.0 |
| 中等波动样本 | 1.5 |
| 异常波动样本 | 3.0 |

也可以做一个更强版本作为附加实验：

| 训练样本类型 | 权重 |
|---|---:|
| 普通波动样本 | 1.0 |
| 中等波动样本 | 2.0 |
| 异常波动样本 | 4.0 |

建议主报告只保留一种权重方案，避免实验过散。若两种都跑，可以把表现更合理的一组作为主结果，另一组作为补充说明。

## 六、评价指标

### 6.1 整体指标

对整体验证集计算：

1. Pearson 相关系数。
2. RMSE。
3. MAE。

### 6.2 分组指标

对普通波动、中等波动、异常波动三组分别计算：

1. 分组 Pearson。
2. 分组 RMSE。
3. 分组 MAE。
4. 该组样本数量和样本占比。
5. 该组平方误差占整体平方误差的比例。
6. 该组绝对误差占整体绝对误差的比例。

其中第 5、6 项很适合支撑结论：

```text
异常波动样本虽然只占验证集约 5%，但可能贡献了显著更高比例的平方误差。
```

## 七、实验流程

### 7.1 数据与验证划分

沿用当前官方任务的时间顺序 Holdout：

```text
前 80% 训练
后 20% 验证
```

要求：

1. 不使用随机划分。
2. 不使用测试集标签。
3. 分组指标只在验证集上统计。
4. 加权训练的权重阈值根据训练集 `abs(label)` 分位数确定。

### 7.2 实验步骤

| 步骤 | 内容 | 验证方式 |
|---|---|---|
| E1 | 读取训练数据，构建与官方任务一致的基础特征 | 特征数量与现有 Ridge/LightGBM 保持一致 |
| E2 | 使用 Ridge 和普通 LightGBM 生成验证集预测 | 整体 Pearson、RMSE、MAE 与现有结果接近 |
| E3 | 按验证集 `abs(label)` 分位数划分普通、中等、异常波动样本 | 输出各组样本数和占比 |
| E4 | 计算各模型在不同波动区间上的分组指标 | 输出分组结果表 |
| E5 | 训练 Weighted LightGBM，并生成验证集预测 | 与普通 LightGBM 对比整体和分组指标 |
| E6 | 统计各波动区间对整体误差的贡献 | 输出误差贡献表 |
| E7 | 生成图表并写入报告分析 | 图表能直接支持结论 |

## 九、报告大纲

### 9.1 任务背景

说明官方任务使用整体 Pearson 评价模型，但金融预测中大幅涨跌样本具有更高分析价值。由于异常波动样本数量少、噪声大，模型可能在整体指标尚可的情况下对异常样本预测较差。

### 9.2 异常波动样本定义

介绍 `abs(label)` 作为波动强度指标，并说明三组分位数划分：

```text
普通波动：0% - 80%
中等波动：80% - 95%
异常波动：95% - 100%
```

同时给出各组样本数量、占比和 `label` 分布。

### 9.3 模型与实验设置

介绍参与对比的模型：

1. Ridge baseline。
2. LightGBM baseline。
3. Weighted LightGBM。

说明 Weighted LightGBM 的权重设置：

```text
普通波动样本权重 1.0
中等波动样本权重 1.5
异常波动样本权重 3.0
```

### 9.4 实验结果

建议放两张表：

第一张表：整体结果。

| 模型 | Pearson | RMSE | MAE |
|---|---:|---:|---:|
| Ridge | 待填 | 待填 | 待填 |
| LightGBM | 待填 | 待填 | 待填 |
| Weighted LightGBM | 待填 | 待填 | 待填 |

第二张表：分组结果。

| 模型 | 波动组 | 样本占比 | Pearson | RMSE | MAE | 平方误差贡献 |
|---|---|---:|---:|---:|---:|---:|
| LightGBM | 普通波动 | 待填 | 待填 | 待填 | 待填 | 待填 |
| LightGBM | 中等波动 | 待填 | 待填 | 待填 | 待填 | 待填 |
| LightGBM | 异常波动 | 待填 | 待填 | 待填 | 待填 | 待填 |
| Weighted LightGBM | 普通波动 | 待填 | 待填 | 待填 | 待填 | 待填 |
| Weighted LightGBM | 中等波动 | 待填 | 待填 | 待填 | 待填 | 待填 |
| Weighted LightGBM | 异常波动 | 待填 | 待填 | 待填 | 待填 | 待填 |

### 9.5 结果分析

重点分析以下方向：

1. 异常波动样本的 RMSE 是否显著高于普通样本。
2. 异常波动样本的误差贡献是否高于其样本占比。
3. Ridge 和 LightGBM 在异常样本上的表现差异。
4. Weighted LightGBM 是否改善异常样本 RMSE、MAE 或 Pearson。
5. 如果加权模型整体 Pearson 下降，说明异常样本预测和整体平均表现之间存在权衡。

### 9.6 局限性

建议如实写：

1. 异常波动样本数量较少，分组指标可能波动较大。
2. 仅用 `abs(label)` 定义异常波动，不能完全代表真实市场风险。
3. 样本加权只改变损失函数关注点，不能从根本上消除金融数据高噪声问题。
4. 测试集没有真实标签，因此异常波动分析主要基于本地验证集完成。

## 十二、最终可写结论模板

如果加权模型改善异常样本：

```text
实验结果表明，普通 LightGBM 在整体验证集上具有较好表现，但异常波动样本上的误差明显更高。
Weighted LightGBM 通过提高大幅波动样本的训练权重，在异常波动区间内取得了更低的 RMSE/MAE，
说明样本加权可以在一定程度上改善模型对极端行情的关注。
但该方法可能牺牲普通波动样本或整体 Pearson，体现出异常样本预测与整体预测性能之间的权衡。
```

如果加权模型没有改善异常样本：

```text
实验结果表明，异常波动样本是模型预测中更困难的部分，其误差水平明显高于普通波动样本。
虽然 Weighted LightGBM 提高了异常样本的训练权重，但未能稳定改善异常波动区间的预测效果，
说明这类样本可能具有更强噪声和更明显的不可预测性，仅通过样本加权难以解决。
后续可以考虑结合更丰富的市场状态特征、时间序列结构或专门的极端行情识别模型。
```
