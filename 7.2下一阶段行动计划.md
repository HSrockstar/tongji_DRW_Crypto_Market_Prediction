## 2. 从冠军方案中提炼出的核心结论

冠军方案最值得借鉴的不是某个固定参数，而是一套特征处理流程：

```text
大量原始匿名特征
→ 按特征间相关性聚类，减少冗余
→ 删除几乎不相关的特征
→ 用多折 CV 和 SHAP 找稳定重要特征
→ 只在精选特征上做线性组合
→ 重新筛选
→ 最后再尝试 AutoEncoder 深度特征
```

对当前项目来说，最重要的判断是：

1. 当前模型基本是在直接拟合全部匿名特征和少量派生特征。
2. 下一阶段不应优先堆模型参数，而应优先建立稳定验证体系。
3. 特征筛选比盲目加入更多特征更重要。
4. 特征合成应在筛选后的小特征集上做，不能直接对 780 个匿名特征暴力组合。
5. AutoEncoder 可以作为后期实验，不适合作为当前第一步。

## 3. 下一阶段总目标

下一阶段的目标不是一次性复现冠军方案，而是把当前项目从“基础模型跑通”推进到“可解释、可验证、可持续迭代”的实验体系。

成功标准：

1. 本地分数从单次 holdout 升级为多折时间 CV。
2. 能得到一份相关性聚类后的代表特征集。
3. 能比较“全量特征”和“聚类降维特征”的模型效果。
4. 能得到一份跨折稳定出现的重要特征列表。
5. 能在 30 到 60 个精选特征上尝试少量线性组合。
6. 每次实验都有明确输出文件、指标和可复现实验记录。

## 4. 推荐执行顺序

### 阶段 1：固定更可靠的本地验证口径

目标：先解决“如何判断一个实验是否真的变好”。

当前单次 holdout 的问题：

```text
前 80% 训练，后 20% 验证
```

这种方式简单，但只能说明模型在最后一段数据上的表现。如果最后 20% 恰好偏难或偏易，实验结论会不稳定。

建议增加时间顺序 CV：

| 验证方式 | 说明 | 建议用途 |
| --- | --- | --- |
| 单次时间 holdout | 当前做法，前 80% 训练，后 20% 验证 | 保留为最终 sanity check |
| blocked CV | 按时间顺序切成多个连续块，多次验证 | 作为主实验评估 |
| walk-forward CV | 每折都用过去预测未来 | 最贴近真实预测任务 |

建议先做 5 折或 6 折时间 CV。每折记录：

| 字段 | 含义 |
| --- | --- |
| `fold` | 第几折 |
| `train_start`、`train_end` | 训练区间 |
| `valid_start`、`valid_end` | 验证区间 |
| `pearson` | 主指标 |
| `rmse`、`mae` | 辅助指标 |
| `feature_count` | 使用特征数量 |
| `model` | 模型名称 |

最终比较时不要只看单个 Pearson，而要看：

```text
Pearson 均值
Pearson 标准差
最后一段 holdout Pearson
```

判断标准：

| 现象 | 解释 |
| --- | --- |
| 均值上升，标准差下降 | 明显更好 |
| 均值上升，标准差大幅上升 | 可能只在部分时间段有效 |
| 均值下降，最后一段上升 | 可能过度贴合近期分布，需要谨慎 |
| 均值不变，特征数大幅下降 | 仍然有价值，因为模型更简单 |

阶段产物：

| 产物 | 内容 |
| --- | --- |
| `outputs/experiments/cv_results.csv` | 多折 CV 明细 |
| `outputs/experiments/cv_summary.csv` | 各模型均值、标准差汇总 |
| `outputs/figures/prediction_task/cv_pearson_by_fold.png` | 不同折分数趋势图 |

### 阶段 2：建立特征分组基线

目标：先弄清楚不同特征组各自有多少贡献。

当前模型大致使用：

```text
5 个公开市场特征
+ 780 个匿名 X 特征
+ 7 个基础市场派生特征
```

建议先跑以下特征组实验：

| 实验 | 特征集 | 目的 |
| --- | --- | --- |
| G1 | 仅 5 个公开市场特征 | 看公开数据是否有独立预测力 |
| G2 | 仅 7 个基础派生市场特征 | 看简单市场派生特征是否有效 |
| G3 | 公开市场特征 + 派生市场特征 | 检验公开市场特征工程收益 |
| G4 | 仅 `X1` 到 `X780` | 看匿名特征主体贡献 |
| G5 | 匿名特征 + 公开市场特征 | 对比是否需要公开字段 |
| G6 | 当前完整特征集 | 当前 baseline |

建议模型：

1. Ridge 作为线性基线。
2. LightGBM 作为非线性主模型。

阶段产物：

| 产物 | 内容 |
| --- | --- |
| `outputs/experiments/feature_group_cv_results.csv` | 每个特征组的每折结果 |
| `outputs/experiments/feature_group_cv_summary.csv` | 每个特征组的均值和标准差 |
| `outputs/figures/feature_effectiveness/feature_group_compare.png` | 特征组对比图 |

判断标准：

1. 如果仅匿名特征明显强于公开市场特征，后续筛选应以 `X` 特征为主。
2. 如果公开市场派生特征有稳定增益，应继续保留。
3. 如果某一组特征只在单折有效，不应过早纳入最终方案。

### 阶段 3：相关性聚类降维

目标：减少 780 个匿名特征中的冗余特征。

操作思路：

```text
取训练数据中的 X1-X780
→ 计算特征间 Pearson 相关性
→ 使用 |corr| > 0.6 作为高度相关阈值
→ 将高度相关的特征聚类
→ 每个聚类选择 1 个 medoid 代表特征
→ 得到代表特征集
```

medoid 的直观含义：

```text
一个簇中最能代表其他特征的真实特征
```

不是新造一个平均特征，而是从原始特征中选一个代表。这样做的好处是：

1. 保留原始字段含义和可追踪性。
2. 降低特征数量。
3. 减少共线性。
4. 让后续 SHAP 和组合特征更可控。

训练方式：

```text
原始完整特征
→ 替换为聚类代表特征
→ 加上公开市场特征和基础派生特征
→ 使用同样的 CV 训练 Ridge 和 LightGBM
→ 与当前完整特征 baseline 对比
```

需要注意：

1. 聚类规则应只在训练折内拟合，避免验证集信息泄露。
2. 如果先做简化版本，可以在完整训练集上生成一次候选聚类表，再用 CV 验证效果。
3. 阈值 `0.6` 是起点，不是定律，可以比较 `0.5`、`0.6`、`0.7`。

阶段产物：

| 产物 | 内容 |
| --- | --- |
| `outputs/experiments/feature_clusters.csv` | 每个特征所属聚类 |
| `outputs/experiments/medoid_features.json` | 聚类代表特征列表 |
| `outputs/experiments/medoid_cv_summary.csv` | 聚类特征集 CV 结果 |

判断标准：

| 结果 | 处理方式 |
| --- | --- |
| 特征数大幅下降，CV 均值不降 | 接受该特征集 |
| CV 均值上升 | 作为后续主特征池 |
| CV 均值小幅下降，但标准差明显下降 | 可作为稳定版本保留 |
| CV 明显下降 | 调整相关性阈值或代表特征选择方式 |

### 阶段 4：目标相关性粗筛

目标：删除明显没有信号的特征，但不永久丢弃。

冠军方案中提到删除：

```text
abs(corr(feature, label)) <= 1e-4
```

对当前项目建议这样使用：

1. 先计算每个特征与 `label` 的 Pearson 相关性。
2. 将绝对相关性极低的特征标记为低优先级。
3. 不要直接永久删除这些特征。
4. 在组合特征和 AutoEncoder 阶段仍可考虑“回收”部分特征。

建议实验：

| 实验 | 特征集 |
| --- | --- |
| C1 | medoid 特征 |
| C2 | medoid 特征中删除 `abs(corr) <= 1e-4` 的特征 |
| C3 | medoid 特征中删除 `abs(corr) <= 5e-4` 的特征 |
| C4 | medoid 特征中删除 `abs(corr) <= 1e-3` 的特征 |

阶段产物：

| 产物 | 内容 |
| --- | --- |
| `outputs/experiments/feature_target_corr.csv` | 特征与目标相关性 |
| `outputs/experiments/low_signal_features.json` | 低相关特征列表 |
| `outputs/experiments/corr_filtered_cv_summary.csv` | 不同阈值筛选结果 |

判断标准：

1. 如果删除低相关特征后均值上升，说明降噪有效。
2. 如果均值变化不大但特征数减少，也可接受。
3. 如果均值明显下降，说明单变量低相关不代表无用，需降低阈值。

### 阶段 5：基于 CV 和 SHAP 的稳定特征筛选

目标：找出跨时间段稳定重要的特征，而不是依赖单次训练结果。

建议流程：

```text
使用阶段 3 或阶段 4 的候选特征集
→ 每一折训练 LightGBM
→ 每折计算 SHAP 或 feature importance
→ 取每折 top-20 或 top-30 特征
→ 统计每个特征出现次数
→ 保留跨折反复出现的特征
```

筛选规则建议：

| 规则 | 含义 |
| --- | --- |
| 出现次数 >= 3/6 | 较宽松，适合探索 |
| 出现次数 >= 4/6 | 较稳健，适合作为主特征集 |
| 出现次数 >= 5/6 | 很严格，可能遗漏有用特征 |

阶段产物：

| 产物 | 内容 |
| --- | --- |
| `outputs/experiments/fold_feature_importance.csv` | 每折特征重要性 |
| `outputs/experiments/stable_features.json` | 稳定特征列表 |
| `outputs/experiments/stable_feature_cv_summary.csv` | 稳定特征集 CV 结果 |

判断标准：

1. 稳定特征集数量建议控制在 20 到 60 个。
2. 如果稳定特征集性能接近全量特征，说明筛选有效。
3. 如果性能更好，后续特征组合应基于这个特征集进行。
4. 如果性能明显下降，说明 top-k 或出现次数阈值过严。

### 阶段 6：精选特征上的线性组合

目标：在 30 到 60 个稳定特征上做少量可控的特征合成。

不建议做：

```text
直接对 780 个匿名特征做所有两两组合
```

原因：

1. 组合数量会爆炸。
2. 训练成本高。
3. 极容易过拟合。
4. 很难解释和复现。

建议只在稳定特征上尝试：

| 组合类型 | 示例 |
| --- | --- |
| 加法 | `a + b` |
| 差值 | `a - b` |
| 比值 | `a / (abs(b) + eps)` |
| 乘积 | `a * b` |

组合候选不要一次性全部加入。建议先按以下规则筛选组合：

1. 只组合稳定特征中的 top 20。
2. 优先组合相关性不太高但都重要的特征。
3. 对每个组合计算与 `label` 的相关性。
4. 只保留相关性排名靠前的少量组合。
5. 加入组合后重新跑 CV。

阶段产物：

| 产物 | 内容 |
| --- | --- |
| `outputs/experiments/synthesized_feature_candidates.csv` | 候选组合特征及其相关性 |
| `outputs/experiments/synthesized_features.json` | 最终保留的组合特征定义 |
| `outputs/experiments/synthesized_feature_cv_summary.csv` | 组合特征实验结果 |

判断标准：

1. 组合特征必须在 CV 均值上有收益，或显著降低波动。
2. 如果只提高单折，不应采用。
3. 每次新增组合特征数量应小于原稳定特征数量，避免失控。

### 阶段 7：可选 AutoEncoder 深度特征

目标：在主线稳定后，尝试使用 AutoEncoder 生成少量压缩特征。

这个阶段暂不作为优先任务。建议满足以下条件后再做：

1. 已经有稳定 CV。
2. 已经有 medoid 特征集。
3. 已经有稳定特征集。
4. 已经完成线性组合实验。

建议方式：

```text
输入：最终精选特征
模型：AutoEncoder
输出：8 个 latent features
再将 8 个 AE 特征加入 Ridge / LightGBM
重新跑 CV
```

风险：

1. AE 容易过拟合。
2. 标准化和训练折处理不当会造成泄露。
3. 对小作业报告来说解释难度更高。

判断标准：

1. CV 均值有稳定提升。
2. 最后一段 holdout 同步提升。
3. AE 特征数量保持较少，例如 4 到 8 个。
4. 报告中能说明 AE 只是压缩特征，不使用 `label` 生成特征。

第一轮只做必要实验，不要过度展开。

| 优先级 | 实验 | 模型 | 验证 | 目的 |

## 6. 近期最小可执行计划

建议接下来按以下顺序推进，每一步都可单独验证。

### 第一步：补时间 CV

任务：

1. 增加 blocked CV 或 walk-forward CV 切分。
2. 用当前完整特征复跑 Ridge 和 LightGBM。
3. 输出每折 Pearson、均值、标准差。

验证：

```text
能得到当前完整特征的 CV 均值和标准差
```

### 第二步：做特征组消融

任务：

1. 分别训练公开市场特征、匿名特征、派生特征、完整特征。
2. 固定模型参数，避免比较不公平。
3. 输出特征组对比表。

验证：

```text
能回答匿名特征、公开市场特征和派生特征各自贡献多少
```

### 第三步：做相关性聚类

任务：

1. 对 `X1` 到 `X780` 计算相关性。
2. 按 `|corr| > 0.6` 聚类。
3. 每簇选择 medoid。
4. 用 medoid 特征集重新训练模型。

验证：

```text
特征数量明显减少，并得到与完整特征可比较的 CV 结果
```

### 第四步：做稳定重要特征筛选

任务：

1. 每折训练 LightGBM。
2. 每折取 top-20 或 top-30 特征。
3. 统计跨折出现次数。
4. 形成稳定特征集。

验证：

```text
得到 20 到 60 个稳定特征，并验证其 CV 表现
```

### 第五步：做少量组合特征

任务：

1. 只基于稳定特征中的 top 20 做组合。
2. 生成少量 `a+b`、`a-b`、`a/b`、`a*b`。
3. 重新跑 CV。

验证：

```text
组合特征带来稳定提升，或证明该方向暂时不值得继续
```

## 7. 决策规则

后续每个新实验都按以下规则判断是否保留：

| 判断项 | 保留条件 |
| --- | --- |
| CV Pearson 均值 | 不低于当前主基线，最好有提升 |
| CV Pearson 标准差 | 不明显变大 |
| 最后一段 holdout | 不能明显劣化 |
| 特征数量 | 如果分数接近，优先选择特征更少的方案 |
| 训练成本 | 如果收益很小但成本大，不进入主线 |
| 可解释性 | 报告中能说明特征来源和筛选逻辑 |

建议不要只因为某一次验证分数变高就采用新方案。金融数据噪声大，单次提升很容易是偶然结果。

## 9. 当前不建议优先做的事

1. 不建议直接大规模调 LightGBM 参数。
2. 不建议直接上复杂神经网络。
3. 不建议对 780 个匿名特征做暴力两两组合。
4. 不建议只看单次 holdout 分数决定方案。
5. 不建议把低相关特征永久删除，应该保留可回收列表。
6. 不建议把 Kaggle public LB 当作唯一依据，本地 CV 仍是主要实验判断依据。

## 11. 推荐优先级总结

最推荐的下一步顺序：
