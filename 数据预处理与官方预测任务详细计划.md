# 数据预处理与官方预测任务详细计划

## 一、计划范围

本计划只针对两个内容：

1. 数据预处理：从 `data/raw/train.parquet`、`data/raw/test.parquet` 和 `data/raw/sample_submission.csv` 出发，建立稳定、可复现、无泄露的数据处理流程。
2. 第一个问题，即 Kaggle 官方短期价格变化回归预测任务：训练基础模型和初步主模型，完成本地验证，并生成符合提交格式的 `submission.csv`。

本计划暂不处理以下内容：

1. 特征有效性与可解释性分析。
2. 不同时间窗口下的模型稳定性分析。
3. PPT、论文报告的完整撰写。
4. 大规模模型融合、深度学习模型或复杂自动调参。

这样做的原因是先把官方预测任务跑通，得到一个可信的本地验证分数和可提交文件，再进入后续扩展任务会更稳。

## 二、当前数据基础

根据当前仓库已有校验结果：

| 文件 | 路径 | 作用 | 已确认信息 |
| --- | --- | --- | --- |
| 训练集 | `data/raw/train.parquet` | 训练和本地验证 | 525886 行，787 列 |
| 测试集 | `data/raw/test.parquet` | 生成最终预测 | 538150 行，786 列 |
| 提交样例 | `data/raw/sample_submission.csv` | 提交格式模板 | 两列：`ID`、`prediction` |

核心字段：

| 类型 | 字段 |
| --- | --- |
| 公开市场特征 | `bid_qty`、`ask_qty`、`buy_qty`、`sell_qty`、`volume` |
| 匿名特征 | `X1` 到 `X780` |
| 目标变量 | `label` |
| 可能存在的索引列 | `__index_level_0__` |

基本处理原则：

1. `train.parquet` 的 `label` 用于训练和本地验证。
2. `test.parquet` 只用于最终预测和提交文件生成，不参与本地验证和调参。
3. 即使测试集中存在 `label` 列，也按测试标签不可用处理，不读取该列作为真实目标。
4. 本地验证必须按行顺序或时间顺序切分，不能随机切分。

## 三、成功标准

完成本阶段后，应具备以下成果：

1. 能用脚本稳定读取原始 parquet 数据。
2. 能完成缺失值、无穷值、数据类型、特征列和目标列的统一处理。
3. 能按时间顺序构造训练集和验证集。
4. 能训练一个基础模型，并得到本地验证指标。
5. 能训练一个初步主模型，并得到本地验证指标。
6. 能生成 `outputs/submissions/submission.csv`。
7. `submission.csv` 与 `sample_submission.csv` 行数一致，列名一致，预测值无缺失、无无穷。
8. 所有关键结果保存到 `outputs/experiments`，方便后续写报告。

## 五、数据预处理详细计划

### 1. 环境与文件检查

目标：确认运行环境和数据文件可用。

执行内容：

1. 运行 `src/check_env.py`，确认依赖库可导入。
2. 运行 `src/verify_data.py --root E:\DRW`，确认三个原始数据文件存在。
3. 检查 `sample_submission.csv` 的列名和行数。
4. 检查 `train.parquet` 和 `test.parquet` 的字段是否一致。

验收标准：

1. 依赖检查通过。
2. 原始数据文件存在。
3. `sample_submission.csv` 包含 `ID`、`prediction` 两列。
4. 测试集预测行数与提交样例行数一致。

### 2. 字段识别与列清理

目标：明确哪些列用于建模，哪些列必须排除。

处理规则：

1. 目标列：`label`。
2. 排除列：`label`、`__index_level_0__`、明显无意义的索引列。
3. 特征列：除排除列外的所有数值列。
4. 公开市场特征：保留原始 5 个字段。
5. 匿名特征：保留 `X1` 到 `X780`。

建议实现逻辑：

```text
target_col = "label"
drop_cols = ["label", "__index_level_0__"]
feature_cols = 所有不在 drop_cols 中的数值列
```

验收标准：

1. 训练集特征列和测试集特征列完全一致。
2. 训练时不会把 `label` 放入特征矩阵。
3. 提交预测时不会依赖测试集 `label`。

### 3. 数据类型压缩

目标：降低内存占用，避免训练时内存压力过大。

处理规则：

1. 将浮点特征统一转换为 `float32`。
2. 若存在整数特征，可在不丢信息的情况下转为较小整数类型，或统一转为 `float32`。
3. `label` 保持 `float32` 或 `float64` 均可；计算指标时可转为 `float64` 保证数值稳定。

验收标准：

1. 单次训练不会因为无意义的数据类型占用导致内存不足。
2. 转换后行数、列数不发生变化。

### 4. 缺失值与无穷值处理

目标：先根据真实数据情况判断是否需要处理，再选择最简单、最适合本题场景的方法。

#### 4.1 实际检查结果

已使用 `polars` 对 `data/raw/train.parquet` 和 `data/raw/test.parquet` 的全部数值列进行扫描，检查内容包括：

1. 空值：`null`
2. 非数字值：`NaN`
3. 正负无穷：`inf`、`-inf`

检查结果如下：

| 文件 | 行数 | 列数 | 数值列数 | `null` 总数 | `NaN` 总数 | `inf/-inf` 总数 | 异常列数 |
| --- | --- | --- | --- | --- | --- | --- | --- |
| `train.parquet` | 525886 | 787 | 786 | 0 | 0 | 0 | 0 |
| `test.parquet` | 538150 | 786 | 786 | 0 | 0 | 0 | 0 |

结论：当前原始训练集和测试集不需要做实际缺失值填补，也不需要删除含缺失或无穷值的样本。

#### 4.2 当前阶段推荐策略

由于实测没有缺失值、NaN 和无穷值，本阶段建议采用“防御性检查 + 不主动填补”的策略：

1. 在 `src/data_preprocessing/inspect_missing_inf.py` 中保留完整检查逻辑，并把检查结果保存到 `outputs/experiments/missing_inf_report.json`。
2. 在 `src/data_preprocessing/preprocess.py` 中保留 `inf/-inf` 到 `NaN` 的防御性转换逻辑，但如果转换后发现任何缺失值，先报错或记录异常，不默认静默填补。
3. 对原始特征不做中位数填补、均值填补或 0 填补，因为当前没有需要填补的位置。
4. 对派生特征的除法统一加入 `eps = 1e-6`，避免分母为 0 时人为制造 `inf/-inf`。
5. 每次生成训练矩阵、验证矩阵和测试矩阵后，都执行一次轻量检查，确认没有新产生的 `NaN`、`inf`、`-inf`。

这样处理的理由：

1. 本题数据是高维匿名金融特征，盲目填补会改变原始特征分布。
2. 当前数据没有缺失和无穷值，额外填补不会带来收益，反而增加不必要的处理步骤。
3. 官方预测任务初步阶段更需要保持流程简单、可复现、低干扰。
4. 防御性检查可以防止后续派生特征或新数据版本引入异常值。

#### 4.3 如果后续发现异常值，可选处理方法

如果后续新增派生特征、切换数据版本或读取方式导致出现缺失值或无穷值，可按以下优先级处理：

| 方法 | 适用场景 | 是否推荐 | 理由 |
| --- | --- | --- | --- |
| 方法一：训练集统计量中位数填补 | 少量缺失，且需要训练 Ridge 等线性模型 | 推荐 | 中位数对极端值不敏感，适合金融数据长尾分布；填补统计量只从训练子集计算，可避免泄露 |
| 方法二：LightGBM 原生缺失值处理 | 只训练 LightGBM，且缺失本身可能有信息 | 可选 | LightGBM 能学习缺失方向，不必人为指定填充值；但 Ridge 不能直接使用 |
| 方法三：0 填补 | 特征本身以 0 表示中性状态，或匿名特征已经标准化 | 谨慎使用 | 简单快速，但可能人为制造大量 0，改变匿名特征分布 |
| 方法四：删除异常行 | 只有极少数样本异常，且异常集中在少量行 | 不作为首选 | 金融数据样本宝贵，删除行会改变时间顺序和验证集分布 |
| 方法五：删除异常列 | 某列缺失率极高或全为异常值 | 仅在必要时使用 | 当前没有这类列；删除匿名特征可能损失有效信号 |

若必须填补，本题初步阶段推荐顺序为：

```text
线性模型：inf/-inf -> NaN -> 训练子集中位数填补 -> 标准化
LightGBM：inf/-inf -> NaN -> 优先让 LightGBM 原生处理，或与线性模型保持一致使用训练子集中位数填补
```

验收标准：

1. `inspect_missing_inf.py` 能输出训练集和测试集的异常值统计。
2. 当前数据检查结果应显示 `null`、`NaN`、`inf/-inf` 总数均为 0。
3. 预处理和派生特征生成后再次检查，确认没有新产生异常值。
4. 如果未来出现异常值，处理策略必须只使用训练子集统计量，不能使用验证集或测试集统计量。

### 5. 基础派生特征

目标：在不引入未来信息的前提下，为官方预测任务增加少量合理特征。

初版只构造当前行可计算的市场派生特征，不构造滚动窗口和滞后特征，避免时间索引不明确时引入复杂度。

建议新增特征：

| 特征名 | 公式 | 含义 |
| --- | --- | --- |
| `book_total_qty` | `bid_qty + ask_qty` | 买卖盘总量 |
| `trade_total_qty` | `buy_qty + sell_qty` | 主动成交总量 |
| `book_imbalance` | `(bid_qty - ask_qty) / (bid_qty + ask_qty + eps)` | 买卖盘不平衡 |
| `trade_imbalance` | `(buy_qty - sell_qty) / (buy_qty + sell_qty + eps)` | 主动买卖不平衡 |
| `volume_per_book_qty` | `volume / (bid_qty + ask_qty + eps)` | 成交量相对挂单规模 |
| `volume_per_trade_qty` | `volume / (buy_qty + sell_qty + eps)` | 成交量相对主动成交规模 |
| `log_volume` | `log1p(max(volume, 0))` | 缓解成交量长尾 |

注意：

1. `eps` 建议设为 `1e-6`。
2. 派生特征只使用当前行字段，不使用未来行。
3. 若某些原始值存在负数，`log1p` 前需要先检查分布；不满足条件时可跳过该特征。

验收标准：

1. 派生特征在训练集和测试集上以同样逻辑生成。
2. 派生特征无缺失、无无穷。
3. 初版特征数增加有限，避免把预处理阶段做成复杂特征工程。

### 6. 标准化策略

目标：让线性模型能稳定训练，同时不影响树模型。

处理规则：

1. Ridge 模型需要标准化，推荐使用 `StandardScaler`。
2. 标准化参数只能在训练子集上 `fit`。
3. 验证集和测试集只使用已拟合的 scaler 做 `transform`。
4. LightGBM 不需要标准化，可以直接使用异常检查通过后的原始数值特征。

验收标准：

1. 当前数据无缺失时，Ridge 模型训练流程为 `StandardScaler + Ridge`。
2. 如果未来检测到缺失值，再按训练子集统计量增加 `SimpleImputer`，流程变为 `SimpleImputer + StandardScaler + Ridge`。
3. LightGBM 模型训练流程可以不使用 scaler。

## 六、本地验证方案

### 1. 主验证切分

目标：构造一个简单、稳定、符合时间序列原则的本地验证集。

建议初版切分：

```text
训练集前 80% 行：本地训练
训练集后 20% 行：本地验证
```

可选增强：

```text
训练集前 79% 行：本地训练
中间 1% 行：gap，不用于训练和验证
最后 20% 行：本地验证
```

初版建议先不加 gap，保证流程简单；如果本地分数异常偏高，再加入 gap 检查。

验收标准：

1. 不使用随机切分。
2. 验证集位于训练集之后。
3. 所有模型复用同一切分。

### 2. 评价指标

主指标：

```text
Pearson 相关系数
```

辅助指标：

```text
RMSE
MAE
```

说明：

1. 官方任务应以 Kaggle Evaluation 页面为准；初版计划暂按 Pearson 相关系数作为主指标。
2. Pearson 关注预测值和真实值的线性相关性，不要求预测值绝对尺度完全一致。
3. RMSE 和 MAE 只作为辅助观察，不作为最终模型选择的唯一依据。

验收标准：

1. 每次实验输出 Pearson、RMSE、MAE。
2. 实验结果保存为 CSV 或 JSON。

## 七、官方预测任务初步建模计划

### 1. 基础模型：Ridge Regression

目标：建立第一个可复现 baseline，确认数据流程、指标计算和提交生成逻辑可用。

输入特征：

1. 所有原始数值特征。
2. 少量基础派生市场特征。

训练流程：

1. 读取训练集。
2. 构造基础派生特征。
3. 按时间顺序切分训练集和验证集。
4. 执行缺失值、NaN、无穷值检查；当前数据检查结果为全 0 时不做填补。
5. 使用训练子集拟合标准化器。
6. 训练 Ridge 模型。
7. 在验证集上预测。
8. 计算 Pearson、RMSE、MAE。
9. 保存模型、scaler、特征列和指标结果。

建议初始参数：

```text
alpha: 1.0
random_state: 42
```

可选小范围参数：

```text
alpha: [0.1, 1.0, 10.0, 100.0]
```

验收标准：

1. Ridge 能完成训练和验证。
2. 本地验证 Pearson 能正常计算，不为 `NaN`。
3. 指标结果写入 `outputs/experiments/official_baseline_results.csv`。

### 2. 初步主模型：LightGBM Regressor

目标：建立一个比线性 baseline 更强的表格模型，用于官方任务初步解答。

输入特征：

1. 所有原始数值特征。
2. 同一套基础派生市场特征。

训练流程：

1. 读取训练集。
2. 构造基础派生特征。
3. 按时间顺序切分训练集和验证集。
4. 执行缺失值、NaN、无穷值检查；当前数据检查结果为全 0 时不做填补。
5. 训练 LightGBM 回归模型。
6. 使用验证集做 early stopping。
7. 输出验证集预测。
8. 计算 Pearson、RMSE、MAE。
9. 保存模型、特征列、参数和验证结果。

建议初始参数：

```text
objective: regression
metric: rmse
learning_rate: 0.03
num_leaves: 63
max_depth: -1
min_data_in_leaf: 50
feature_fraction: 0.8
bagging_fraction: 0.8
bagging_freq: 1
lambda_l1: 0.0
lambda_l2: 1.0
num_boost_round: 5000
early_stopping_rounds: 100
seed: 42
```

说明：

1. 虽然训练目标是 RMSE，但模型选择时看验证 Pearson。
2. 初版不做大规模调参，只确保流程跑通并获得合理分数。
3. 如果训练过慢，可先减少 `num_boost_round` 或使用部分数据调试流程，再用全量数据正式训练。

验收标准：

1. LightGBM 能完成训练和验证。
2. 验证 Pearson 高于或至少接近 Ridge baseline。
3. 指标结果写入 `outputs/experiments/official_lgbm_results.csv`。
4. 模型保存到 `models/official_lgbm.txt` 或 `models/official_lgbm.pkl`。

### 3. 初步模型选择

目标：从 Ridge 和 LightGBM 中选出一个用于首次提交。

选择规则：

1. 优先选择验证 Pearson 更高的模型。
2. 如果两个模型 Pearson 接近，优先选择训练更稳定、提交预测分布更合理的模型。
3. 如果 LightGBM 明显过拟合，先使用 Ridge 或降低 LightGBM 复杂度。

需要检查的预测分布：

1. 验证集预测均值、标准差、最小值、最大值。
2. 验证集真实 `label` 的均值、标准差、分位数。
3. 测试集预测是否出现极端异常值。

验收标准：

1. 形成 `outputs/experiments/official_model_compare.csv`。
2. 明确记录首次提交使用哪个模型和原因。

## 八、提交文件生成计划

目标：生成符合 Kaggle 要求的 `submission.csv`。

生成流程：

## 十一、风险点与处理策略

| 风险 | 表现 | 处理策略 |
| --- | --- | --- |
| 内存不足 | 读取或训练时崩溃 | 使用 `float32`、减少复制、先调试小样本 |
| 时间泄露 | 本地分数异常偏高 | 坚持时间顺序切分，必要时加入 gap |
| 测试集标签误用 | 不真实的本地分数 | 测试集只用于预测，不读取其 `label` 作为目标 |
| Pearson 为 `NaN` | 预测值常数或方差为 0 | 检查模型输出分布，避免常数预测 |
| 预测极端值过多 | 提交结果不稳定 | 检查预测分布，必要时做温和裁剪或标准化后处理 |
| 特征列不一致 | 提交时报错 | 固化训练阶段的 `feature_cols` 并在预测阶段复用 |
