# DRW 加密货币市场预测大作业解题方法计划

## 一、题目内容概述

本题来源于 Kaggle 竞赛 `DRW - Crypto Market Prediction`，核心任务是基于加密货币市场的分钟级历史数据、公开市场变量和 DRW 提供的匿名化生产特征，预测未来短期价格变化对应的连续型目标变量 `label`。从建模角度看，这是一个高维表格数据上的时间序列回归问题，数据具有金融数据常见的高噪声、非平稳、低信噪比和潜在时间泄露风险。

大作业不仅要求完成官方预测任务，还要求在官方任务基础上完成两个扩展分析任务：

1. 特征有效性与可解释性分析：比较公开市场特征、匿名特征、派生特征和筛选后重要特征对预测效果的贡献，并解释模型主要依赖哪些特征或特征组。
2. 不同时间窗口下的模型稳定性分析：在不同训练窗口、验证窗口或时间切分方式下比较模型表现，分析模型是否受时间分布变化影响。

最终需要形成一套可复现的实验流程，并提交代码、论文格式报告、PPT 和 8 分钟左右的汇报视频。

## 二、已知数据与基本假设

根据当前仓库的校验结果，数据规模如下：

| 文件 | 作用 | 当前校验结果 |
| --- | --- | --- |
| `data/raw/train.parquet` | 训练集 | 525886 行，787 列 |
| `data/raw/test.parquet` | 测试集或提交集 | 538150 行，786 列 |
| `data/raw/sample_submission.csv` | 提交模板 | 包含 `ID`、`prediction` 两列 |

主要字段包括：

| 字段类型 | 字段 |
| --- | --- |
| 公开市场特征 | `bid_qty`、`ask_qty`、`buy_qty`、`sell_qty`、`volume` |
| 匿名化特征 | `X1` 到 `X780` |
| 目标变量 | `label` |
| 可能需要删除的索引列 | `__index_level_0__` |

本计划采用以下假设和边界：

1. 以 `train.parquet` 内的时间顺序或行顺序构造本地训练集和验证集，不做随机切分，避免未来信息泄露。
2. 即使本地 `test.parquet` 中出现 `label` 列，也按题目要求将测试集视为无真实标签数据；测试集只用于生成提交文件，不用于训练、验证或调参。
3. 题目要求以 Kaggle 官方评价指标为主。本地实验建议优先使用 Pearson 相关系数作为主指标，并辅以 RMSE 或 MAE 观察误差规模；正式报告中应再核对 Kaggle 官方 Evaluation 页面。
4. 先完成稳健、可复现的传统机器学习路线。深度学习模型不是本题的必要起点，只有在基础实验充分后才作为扩展尝试。

## 三、总体解题路线

建议采用“先跑通，再比较，再解释”的路线：

1. 数据读取与 EDA：确认字段、缺失值、异常值、目标分布、特征分布和 train/test 分布差异。
2. 时间顺序验证集划分：建立固定的本地评估方案，先获得可信的 baseline。
3. 基础模型训练：使用 Ridge 或 ElasticNet 作为基础模型，建立最低复杂度对照。
4. 主要模型训练：使用 LightGBM 作为主模型，必要时补充 XGBoost 或 CatBoost。
5. 官方任务提交：基于最佳验证方案训练最终模型，生成 `submission.csv`。
6. 特征有效性分析：围绕不同特征组做消融实验和重要性分析。
7. 时间窗口稳定性分析：设计不少于三组时间窗口实验，比较模型随时间变化的表现。
8. 结果整理：输出表格、图表、报告章节和 PPT 汇报材料。

这条路线的优点是工程风险低、结果容易复现，也能直接覆盖题目中“预测、解释、稳定性”三类要求。

## 四、具体解题计划

### 1. 数据理解与探索性分析

目标：弄清数据结构、字段类型、目标变量分布和可能的数据质量问题。

具体工作：

1. 使用 `polars` 或 `pyarrow` 读取 parquet 文件，避免一次性高内存复制。
2. 统计训练集和测试集的行数、列数、字段类型、缺失率、常数列、重复列。
3. 分析 `label` 的均值、标准差、分位数、极端值比例。
4. 分析 5 个公开市场特征的分布，并构造简单市场状态指标。
5. 计算部分特征与 `label` 的 Pearson、Spearman 相关性，初步识别高信号特征。
6. 比较 train/test 的特征分布，标记分布差异较大的字段。

建议输出：

| 输出文件 | 内容 |
| --- | --- |
| `outputs/eda/data_summary.csv` | 数据规模、字段类型、缺失率 |
| `outputs/eda/label_distribution.png` | 目标变量分布 |
| `outputs/eda/top_feature_correlation.csv` | 特征与目标变量相关性排名 |
| `outputs/eda/train_test_shift.csv` | 训练集和测试集分布差异统计 |

验证标准：能够在报告中清楚说明数据字段、目标变量、缺失处理策略和主要数据特征。

### 2. 本地验证方案设计

目标：建立可信的本地评估方式，避免随机切分导致时间泄露。

建议主验证方案：

1. 按训练集原始顺序切分，前 80% 作为训练集，后 20% 作为验证集。
2. 主指标使用 Pearson 相关系数。
3. 辅助指标记录 RMSE、MAE，便于观察预测误差规模。
4. 若担心相邻样本间存在强时间依赖，可在训练集和验证集之间预留一个小的 gap，例如 1000 行，作为防泄露缓冲。

Pearson 相关系数可作为本地主要评价指标：

```text
corr(y, y_pred) = cov(y, y_pred) / (std(y) * std(y_pred))
```

注意事项：

1. 常数预测无法产生有效 Pearson 相关系数，因此“均值预测”只适合作为 RMSE 参照，不适合作为主要 baseline。
2. 所有模型和特征组实验必须使用同一套验证切分，否则结果不可比较。
3. 调参只能看训练集内部切分或本地验证集，不能利用测试集真实信息。

验证标准：形成一个固定的 `train_idx`、`valid_idx` 或切分函数，后续所有实验复用。

### 3. 特征处理方案

目标：构造一套简单、可解释、不会引入未来信息的特征处理流程。

基础处理：

1. 删除目标列 `label` 和无建模意义的索引列 `__index_level_0__`。
2. 将数值列统一转换为 `float32`，降低内存占用。
3. 对线性模型使用中位数或 0 填补缺失值，并进行标准化。
4. 对 LightGBM 可保留缺失值，或使用统一填补策略保证实验可重复。

特征分组：

| 特征组 | 字段或构造方式 | 用途 |
| --- | --- | --- |
| 公开市场特征 | `bid_qty`、`ask_qty`、`buy_qty`、`sell_qty`、`volume` | 检验公开市场变量是否有独立预测力 |
| 匿名特征 | `X1` 到 `X780` | 检验 DRW 生产特征的主要贡献 |
| 派生市场特征 | 买卖盘不平衡、成交方向不平衡、成交量比例、`log1p(volume)` 等 | 增强公开市场变量表达能力 |
| 筛选特征 | 相关性 Top K、重要性 Top K 或低缺失率特征 | 降低噪声和过拟合风险 |

建议派生特征：

| 特征 | 示例公式 | 含义 |
| --- | --- | --- |
| 买卖盘总量 | `bid_qty + ask_qty` | 市场挂单规模 |
| 买卖盘不平衡 | `(bid_qty - ask_qty) / (bid_qty + ask_qty + eps)` | 买卖盘压力差 |
| 主动成交不平衡 | `(buy_qty - sell_qty) / (buy_qty + sell_qty + eps)` | 主动买卖方向差 |
| 成交量强度 | `volume / (bid_qty + ask_qty + eps)` | 成交活跃度 |
| 对数成交量 | `log1p(volume)` | 缓解长尾分布 |

验证标准：每个特征集都能明确说明来源、数量、处理方式和参与实验的范围。

### 4. 基础模型与主要模型

目标：满足题目中“至少一个基础模型和一个主要模型”的要求，并建立合理对比。

建议模型路线：

| 模型角色 | 建议模型 | 原因 |
| --- | --- | --- |
| 基础模型 | Ridge Regression 或 ElasticNet | 训练快、结果稳定、适合高维数值特征 |
| 主要模型 | LightGBM Regressor | 适合表格数据，能处理非线性关系，并提供特征重要性 |
| 可选扩展 | XGBoost、CatBoost 或简单加权融合 | 在主线完成后用于提高分数 |

基础模型实验：

1. Ridge 使用市场特征。
2. Ridge 使用匿名特征。
3. Ridge 使用全部特征。
4. 比较不同正则强度下的 Pearson 和 RMSE。

主要模型实验：

1. LightGBM 使用全部基础特征。
2. LightGBM 使用基础特征加派生特征。
3. LightGBM 使用筛选后的 Top K 特征。
4. 使用 early stopping 控制过拟合。
5. 记录随机种子、参数、训练时间和验证指标。

建议初始 LightGBM 参数方向：

```text
objective = regression
learning_rate = 0.03 到 0.05
num_leaves = 31 到 127
feature_fraction = 0.7 到 0.9
bagging_fraction = 0.7 到 0.9
lambda_l1/lambda_l2 = 适度正则
random_state = 42
```

验证标准：得到基础模型和主要模型的对比表，且所有结果可由脚本重新生成。

### 5. 官方预测任务完成方案

目标：生成符合 Kaggle 格式的预测文件，并在报告中展示官方任务完成情况。

执行步骤：

1. 使用固定验证方案比较 Ridge、LightGBM 和可选模型。
2. 选择验证 Pearson 最高且稳定性较好的模型作为最终模型。
3. 根据时间窗口稳定性实验决定最终训练数据范围：
   - 如果最近窗口明显更好，使用较近训练窗口训练最终模型。
   - 如果全量训练更稳定，使用全量训练集训练最终模型。
4. 对 `test.parquet` 生成预测值。
5. 读取 `sample_submission.csv`，保持 `ID` 不变，只替换 `prediction`。
6. 输出 `outputs/submissions/submission.csv`。
7. 若 Kaggle 可提交，保存提交成绩截图；若不能提交，保存本地验证结果截图。

验证标准：

1. `submission.csv` 行数与 `sample_submission.csv` 一致。
2. `submission.csv` 只包含 `ID` 和 `prediction` 两列。
3. `prediction` 无缺失值、无无穷值。
4. 报告中包含最终模型参数、验证结果和提交或本地验证截图。

### 6. 特征有效性与可解释性分析

目标：回答“哪些特征或特征组真正有用”。

建议消融实验：

| 实验编号 | 特征设置 | 目的 |
| --- | --- | --- |
| F1 | 仅公开市场特征 | 检验公开变量独立预测能力 |
| F2 | 仅匿名特征 | 检验匿名特征贡献 |
| F3 | 公开市场特征 + 匿名特征 | 作为完整基础特征对照 |
| F4 | 公开市场特征 + 派生市场特征 | 检验派生特征是否提升公开变量 |
| F5 | 全部特征 + 派生特征 | 检验完整特征工程效果 |
| F6 | Top K 重要特征 | 检验降噪和特征筛选效果 |

实验原则：

1. 使用同一验证切分。
2. 优先固定模型参数，只改变特征集合。
3. 若对每个特征集都重新调参，应在报告中说明，否则比较不公平。
4. 同时记录 Pearson、RMSE、特征数量和训练时间。

解释性分析方法：

| 方法 | 用途 |
| --- | --- |
| 相关性排名 | 初步判断单变量线性关系 |
| LightGBM gain importance | 判断模型分裂中贡献较大的特征 |
| Permutation Importance | 观察打乱单个特征后性能下降程度 |
| SHAP 抽样分析 | 可选，用于解释局部和整体贡献，样本量不宜过大 |

建议图表：

1. 不同特征组的 Pearson 对比柱状图。
2. Top 30 特征重要性图。
3. 公开特征、匿名特征、派生特征的贡献对比表。
4. 去除 Top K 特征后的性能下降图。

验证标准：报告能够明确回答以下问题：

1. 公开市场特征单独使用时是否有预测力。
2. 匿名特征是否明显提升模型表现。
3. 派生特征是否带来额外收益。
4. 筛选特征是否能在减少特征数的同时保持性能。
5. 重要性结果是否与消融实验结论一致。

### 7. 不同时间窗口下的模型稳定性分析

目标：研究模型性能是否随时间变化，评估模型对金融非平稳性的敏感程度。

建议至少做三类窗口实验：

| 实验类型 | 设计方式 | 分析重点 |
| --- | --- | --- |
| 固定训练窗口 | 用固定长度历史数据训练，验证后续固定长度窗口 | 不同市场阶段表现是否一致 |
| 扩展训练窗口 | 训练集从早期开始逐步扩大，验证后续窗口 | 更多历史数据是否有帮助 |
| 滚动训练窗口 | 训练窗口和验证窗口一起向后滚动 | 最近数据是否比早期数据更有效 |

示例切分方案可以按行顺序分为 5 个连续区间 `B1` 到 `B5`：

| 实验编号 | 训练区间 | 验证区间 | 窗口类型 |
| --- | --- | --- | --- |
| T1 | `B1` | `B2` | 固定窗口 |
| T2 | `B2` | `B3` | 固定窗口 |
| T3 | `B3` | `B4` | 固定窗口 |
| T4 | `B1 + B2` | `B3` | 扩展窗口 |
| T5 | `B1 + B2 + B3` | `B4` | 扩展窗口 |
| T6 | `B2 + B3 + B4` | `B5` | 滚动窗口 |

实际执行时可以根据数据规模和训练耗时调整区间数量，但必须满足题目要求的不少于三组时间窗口实验。

建议统计指标：

1. 每个窗口的 Pearson、RMSE、MAE。
2. 各窗口 Pearson 的均值、标准差、最大值、最小值。
3. 最优窗口和最差窗口的差异。
4. 不同窗口下 Top 特征重要性的一致性。

建议图表：

1. 窗口编号与 Pearson 的折线图。
2. 固定窗口、扩展窗口、滚动窗口的结果对比表。
3. 不同窗口下模型重要特征重合率图。
4. 验证集目标分布随时间变化的图。

验证标准：报告能够说明模型是否稳定、是否更依赖近期数据、是否存在明显时间分布变化。

## 七、优先级与实施顺序

建议按以下顺序执行：

1. 第一优先级：固定验证集、Ridge baseline、LightGBM 主模型、`submission.csv`。
2. 第二优先级：特征组消融实验和特征重要性分析。
3. 第三优先级：时间窗口稳定性实验。
4. 第四优先级：可选模型融合、SHAP 抽样解释、更多调参。

不建议一开始投入复杂神经网络或大规模自动调参。原因是本题评分和报告都强调可复现、可分析和可解释，传统表格模型更容易快速建立可靠结果，也更适合支撑两个扩展任务。

## 八、最终成功标准

项目完成时应满足以下标准：

1. 代码能够从原始 parquet 文件复现实验结果。
2. 至少包含一个基础模型和一个主要模型。
3. 本地验证采用时间顺序切分，并记录主指标 Pearson。
4. 生成符合 `sample_submission.csv` 格式的提交文件。
5. 完成公开市场特征、匿名特征、派生特征和筛选特征的对比实验。
6. 至少完成三组时间窗口稳定性实验。
7. 报告包含必要图表、实验表格和客观分析。
8. PPT 能在 8 分钟左右讲清楚任务、方法、结果、创新点和不足。
