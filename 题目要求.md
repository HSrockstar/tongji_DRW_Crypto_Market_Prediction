# 大作业题目要求

## 题目名称

基于 DRW 加密货币市场数据的短期价格预测、特征有效性分析与时间稳定性研究

## 一、选题来源

选题链接： https://www.kaggle.com/competitions/drw-crypto-market-prediction/overview

本项目选题来源于 Kaggle 竞赛 **DRW - Crypto Market Prediction**。该竞赛要求参赛者基于加密货币市场的分钟级历史数据以及 DRW 提供的生产特征数据，建立模型预测未来短期加密货币价格变化。

根据课程大作业要求，本组选取该 Kaggle 官方题目作为基础任务，并在此基础上自行命题两个扩展任务，分别为：

1. 特征有效性与可解释性分析。
2. 不同时间窗口下的模型稳定性分析。

本项目应同时完成官方预测任务和上述两个扩展任务，并以代码、论文格式报告、PPT 及 8 分钟左右 PPT 报告视频的形式提交。

## 二、项目目标

本项目的总体目标是围绕加密货币短期价格变化预测问题，建立完整的数据建模、实验评估和模型分析流程。项目需完成以下三类任务。

第一，完成 Kaggle 官方预测任务。项目应基于竞赛数据建立预测模型，对未来短期加密货币市场价格变化进行回归预测，并按照官方提交格式生成预测结果。若 Kaggle 平台允许提交，应在报告中呈现提交结果或成绩截图。若平台提交受限，应在报告中说明原因，并呈现本地验证结果。

第二，完成特征有效性与可解释性分析。项目需围绕公开市场特征、匿名特征及构造特征的预测价值展开定量分析，说明不同类型特征对模型预测性能的贡献，并对模型结果进行必要的解释性分析。

第三，完成不同时间窗口下的模型稳定性分析。项目需考虑金融时间序列数据的时间依赖性和非平稳性，比较模型在不同训练窗口、验证窗口或时间切分方式下的性能变化，评估模型在不同市场阶段中的稳定性。

## 三、数据说明

项目使用 Kaggle 竞赛 **DRW - Crypto Market Prediction** 官方提供的数据。数据主要包括训练集、测试集和提交样例文件。

训练集包含加密货币市场的分钟级历史观测数据、若干公开市场变量、若干匿名化特征以及目标变量。公开市场变量可反映买卖盘、成交量和交易方向等市场状态。匿名化特征为竞赛方提供的生产特征，其具体金融含义不公开，但可能包含与市场微观结构、订单流、流动性或价格压力相关的信息。

测试集用于生成最终提交结果。测试集中不提供真实目标变量。项目应严格避免使用任何测试集未来信息、外部泄露数据或不符合竞赛规则的数据来源。

## 四、任务一：官方任务，短期价格变化回归预测

### 4.1 任务定义

官方任务为回归预测任务。给定某一时刻的市场特征，模型需要预测未来短期加密货币价格变化对应的连续型目标变量。

设样本特征为 (X_i)，目标变量为 (y_i)，模型输出为 (\hat{y}_i)。项目需建立映射关系：

[
f(X_i)=\hat{y}_i
]

其中，(\hat{y}_i) 为模型对未来短期价格变化的预测结果。

### 4.2 基本要求

项目至少应包含一个可复现实验流程，包括数据读取、数据预处理、特征处理、模型训练、模型验证、结果预测和提交文件生成。

项目应至少设置一个基础模型和一个主要模型，用于构成基本对比。模型选择不限，但应与数据类型和任务目标相匹配。模型训练过程应记录关键参数、训练设置和验证结果。

### 4.3 评价要求

官方任务应以 Kaggle 竞赛评价指标为主要评价依据。若进行本地验证，应采用与官方目标一致或相近的评价指标，并说明本地验证集的构造方式。

报告中应呈现不同模型或不同实验设置下的预测结果对比，包括但不限于：

1. 基础模型结果。
2. 主要模型结果。
3. 不同特征设置下的模型结果。
4. 最终提交结果或本地验证结果。
5. 结果截图，包括 Kaggle 提交截图或本地实验结果截图。

## 五、任务二：特征有效性与可解释性分析

### 5.1 任务定义

该任务为本项目的自命题扩展任务之一。其目标是分析不同特征在加密货币短期价格变化预测中的有效性，并对模型预测结果进行解释。

该任务关注以下问题：

1. 公开市场特征是否具有独立预测价值。
2. 匿名化特征对模型性能的贡献程度如何。
3. 不同特征组对预测结果的影响是否存在明显差异。
4. 模型主要依赖哪些变量或变量组进行预测。
5. 特征贡献是否具有稳定性和可解释性。

### 5.2 分析对象

特征有效性分析应至少覆盖以下几类特征：

1. 官方提供的公开市场特征。
2. 官方提供的匿名化特征。
3. 经合理处理或构造得到的派生特征。
4. 经过筛选后的重要特征集合。

项目应明确每一类特征的来源、数量、处理方式和参与实验的范围。

### 5.3 实验要求

项目需通过对比实验说明不同特征集合对模型性能的影响。报告中应清楚呈现特征集合变化前后模型性能的差异。

项目需包含至少一种模型解释或特征贡献分析结果。解释性分析应服务于任务本身，避免仅展示无实质解释的图表。

该任务的结果应能够回答：

1. 哪些特征或特征组对预测结果更重要。
2. 去除部分特征后模型性能是否明显下降。
3. 仅使用公开市场特征时模型是否仍具有预测能力。
4. 匿名化特征是否显著提升模型表现。
5. 特征重要性结果与模型预测效果之间是否一致。

### 5.4 输出要求

该任务在报告中应至少包含：

1. 特征分类说明。
2. 不同特征集合下的模型对比结果。
3. 特征重要性或解释性分析图表。
4. 对主要特征贡献的客观分析。
5. 对匿名特征可解释性限制的说明。

## 六、任务三：不同时间窗口下的模型稳定性分析

### 6.1 任务定义

该任务为本项目的第二个自命题扩展任务。其目标是研究模型在不同时间窗口下的预测表现，分析加密货币市场数据中可能存在的时间分布变化，并评估模型在不同市场阶段中的稳定性。

该任务关注以下问题：

1. 模型在不同时间段上的预测性能是否稳定。
2. 使用较早数据训练和使用较近数据训练时，模型表现是否存在差异。
3. 训练窗口长度变化是否影响模型预测能力。
4. 验证窗口变化是否导致评价结果明显波动。
5. 模型是否存在对特定时间段过拟合的问题。

### 6.2 时间切分要求

项目应遵循时间序列数据的基本约束，避免随机划分造成未来信息泄露。训练集和验证集的划分应保持时间顺序。

项目应设计不少于三组时间窗口实验。时间窗口可以包括固定窗口、扩展窗口或滚动窗口，但每一组实验均需明确训练区间、验证区间和样本范围。

报告中需说明时间窗口划分的依据，并保证不同时间窗口之间具有可比较性。

### 6.3 实验要求

项目需在不同时间窗口设置下训练或验证模型，并比较模型性能的变化。实验结果应能够反映模型对时间变化的敏感程度。

该任务应重点呈现：

1. 不同训练窗口下的模型性能。
2. 不同验证窗口下的模型性能。
3. 模型性能随时间变化的趋势。
4. 最优时间窗口设置及其稳定性。
5. 模型在不同市场阶段中的表现差异。

### 6.4 输出要求

该任务在报告中应至少包含：

1. 时间窗口划分方案。
2. 各时间窗口下的实验结果表。
3. 模型性能随时间变化的可视化结果。
4. 对模型稳定性和时间分布变化的分析。
5. 对实验局限性的说明。

## 七、实验与结果呈现要求

项目报告应包含完整的实验设计和结果呈现。实验结果需真实、可复现，并与代码结果保持一致。

报告中的实验结果至少应包括：

1. 官方回归预测任务的模型对比结果。
2. 特征有效性与可解释性分析结果。
3. 不同时间窗口下的模型稳定性分析结果。
4. 最终模型的预测结果。
5. Kaggle 提交成绩截图或本地验证截图。
6. 主要实验图表和必要文字说明。

所有图表应具有明确标题、坐标说明和必要注释。结果分析应保持客观，不夸大模型效果。若模型表现有限，应说明可能原因。

总结本项目完成的主要工作、实验结论、创新点和不足之处。
